The Human-Baxter Collaboration Framework  v2.0
The Human-Baxter collaboration framework is a modular, easy to modify and adapt framework for collaborative experiments with human collaborators and a Baxter research robot.

1 # Usage of the Human-Baxter Collaboration Framework
2 
3 Let us introduce the usage of the Human-Baxter collaboration framework using the example of the distributed pick-and-place scenario.
4 For first-time usage, we need to
5 - download the pre-trained neural networks,
6 - perform the calibration routines and
7 - run the experiment.
8 
9 Otherwise it is sufficient to
10 - run the experiment.
11 
12 Note that it might make sense to repeat some parts of the calibration every once in a while.

40 $ roslaunch kinect2_bridge kinect2_bridge.launch

46

61 ### Assemble the Hand Mount for the External Calibration
62 For the external calibration of the Kinect V2 sensor relative to the Baxter robot a hand-mounted calibration pattern is used.
63 The hand mount has been designed in [OpenSCAD](http://www.openscad.org/) to replace the fingers of the electrical gripper.
64 The original CAD file and the derived STL file can be found in `$WS_HBCF/models/mount`.
65 
66 3D print the hand mount.
67 Additionally, you will need a flat piece of wood or acrylic glass of about 21 x 25 cm, a drill and some screws.
68 
69 Print the [calibration pattern](http://docs.opencv.org/2.4.13/_downloads/acircles_pattern.png) and make sure that its dimensions are preserved by the printer.
70 When gluing the patten onto the mount (after it has been screwed to the hand of the robot), make sure that it is completely flat!

86 ## Calibration
87 
88 The calibration performed for the distributed pick-and-place scenario consists of three parts:
89 
90 1. Estimate the table height in Baxter's base coordinates.
91 1. Obtain reference images of the (empty) table to find empty spots for placing objects.
92 1. Estimate the external calibration (the pose (= position and orientation) of the Kinect V2 sensor relative to Baxter's base coordinates).
93 
94 If you run `rosrun baxter_pick_and_place demonstration.py` the program will guide you through the first two calibration tasks (table height and table view).
95 
96 If you run the experiment in simulation mode (see below), the external calibration basically comes for free.
97 Otherwise, an estimate for the external parameters can be obtained, e.g., using the algorithm by Tsai and Lenz (1989).
98 In this case, attach the previously prepared and assembled hand mounted calibration pattern to one of Baxter's limbs.
99 Plug in your Kinect V2 sensor and position it such that it has a clear view of Baxter and the task space.
100 In both cases, run
101 ```bash
102 $ cd $WS_HBCF
103 $ . baxter.sh [sim]
104 $ rosrun baxter_pick_and_place calibrate_external.py
105 ```
106 to perform the external calibration.
107 
108 The three calibration routines will create files `table_height.npz`, `table_view.npz` and `external_parameters.npz` in `$WS_HBCF/data/setup`.
109 To repeat a calibration step (e.g., when lighting conditions change drastically (`table_view.npz`) or when switching from simulation to the real robot (`external_calibration.npz`)) manually delete the corresponding file and run the `calibrate_external.py` or `demonstration.py` script again.

112 ## Run the Experiment
113 
114 Using the Kinect sensor requires some preparation:
115 - If you are intending to use the Kinect V2 on Windows, make sure to plug it in and run the ELTE Kinect Windows tool on the Windows machine.
116 - If you want to use the Kinect V2 on Ubuntu using ROS, plug it into your Ubuntu machine and run the kinect2_bridge by typing
117 ```bash
118 $ cd $WS_HBCF
119 $ . baxter.sh
120 $ rosrun kinect2_bridge kinect2_bridge
121 ```
122 - If you are using the simulated Kinect, the launch script (see below) will take care of the details.
123 
124 
125 To follow the progress of the experiment visual information is published to a ROS image topic.
126 To show the visualization, type (in two terminals with initialized SDK environment) `rosrun baxter_pick_and_place inspection.py` and `rosrun image_view image_view image:=/visualization/image`.

129 ### On the Robot
130 
131 To run the pick-and-place experiment, initialize your SDK environment and `rosrun` the experiment.
132 That is, do
133 ```bash
134 $ cd $WS_HBCF
135 $ . baxter.sh

140 $ cd $WS_HBCF
141 $ . baxter.sh
142 $ rosrun baxter_pick_and_place instruct.py
143 ```
144 and follow the instructions in the terminal.

147 ### In Simulation
148 
149 To start up the simulation environment (Gazebo) and run the demonstration,
150 initialize your SDK environment in simulation mode, `roslaunch` the simulator
151 and `rosrun` the experiment.
152 That is, in a terminal do
153 ```bash
154 $ cd $WS_HBCF
155 $ . baxter.sh sim
156 $ roslaunch baxter_pick_and_place simulation.launch depth_external:=true
157 ```
158 In another terminal do
159 ```bash
160 $ cd $WS_HBCF
161 $ . baxter.sh sim
162 $ rosrun baxter_pick_and_place demonstration.py
163 ```
164 And in a third terminal run the instruction module by typing
165 ```bash
166 $ cd $WS_HBCF
167 $ . baxter.sh sim
168 $ rosrun baxter_pick_and_place instruct.py
169 ```
170 Now follow the instructions in the third terminal.
171 
172 Note: The `roslaunch` files have parameters to modify their behavior. Please
173 have a look at the files for more information.

176 ### Experiment Convenience Launch File
177 
178 There also is a launch file that collects the separate steps above into
179 one file.
180 To [start up the simulation environment and] run the experiment, do
181 ```bash
182 $ cd $WS_HBCF
183 $ . baxter.sh [sim]
184 $ roslaunch baxter_pick_and_place demonstration.launch [gazebo:=true] [record_vis:=true]
185 ```
186 where the optional `gazebo` flag indicates whether the simulation environment needs to be started and `record_vis` records the visualization topic into a rosbag file and stores it in /tmp/pnp-vis*.
187 In another terminal run the instruction module by typing
188 ```bash
189 $ cd $WS_HBCF
190 $ . baxter.sh [sim]
191 $ rosrun baxter_pick_and_place instruct.py
192 ```
