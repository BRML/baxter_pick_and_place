1 # Introducing the Human-Baxter Collaboration Framework
2 
3 This repository contains [BRML](https://brml.org/brml/)'s Human-Baxter collaboration framework.
4 It was implemented as part of the [EIT CPS for Smart Factories project](http://dfki.de/smartfactories/), in collaboration with the [Neural Information Processing Group](http://nipg.inf.elte.hu/) at ELTE, Budapest, Hungary and [DFKI](http://dfki.de/web) Saarbruecken, Germany.

7 ## Overview
8 
9 The Human-Baxter collaboration framework aims at being a modular, easy to modify and adapt, framework for collaborative experiments with human collaborators and a [Baxter research robot](http://www.rethinkrobotics.com/research-education/).
10 
11 The distributed pick-and-place scenario integrates the Baxter robot, the Microsoft Kinect V2 sensor and deep neural networks to detect, pick up and place objects.
12 Three types of experiments are possible:
13 - picking and placing an object on a table,
14 - handing over an object to a human collaborator and
15 - taking over an object from a human collaborator.

18 ## Dependencies and Requirements
19 
20 Two possibilities to use the Human-Baxter collaboration framework exist.
21 You either need the open source [Baxter simulator](http://sdk.rethinkrobotics.com/wiki/Baxter_Simulator) or access to a [Baxter research robot](http://www.rethinkrobotics.com/research-education/).
22 
23 The Kinect V2 can be interfaced either on Ubuntu/ROS via the [iai_kinect2](https://github.com/code-iai/iai_kinect2) package (no skeleton tracking) or via a web socket connection and the ELTE Kinect Windows tool running on a Windows machine.
24 
25 The framework heavily builds upon the [Baxter SDK](https://github.com/RethinkRobotics) and depends on customized versions of [baxter_interface](https://github.com/BRML/baxter_interface.git) and [baxter_common](https://github.com/BRML/baxter_common.git) from [Rethink Robotics](http://www.rethinkrobotics.com/).
26 For the simulation in Gazebo, the [depth_sensors](https://github.com/BRML/depth_sensors.git) package is utilized.
27 For image processing we rely on the [OpenCV](http://opencv.org/) library.
28 
29 
30 The framework has been tested with ROS Indigo on Ubuntu 14.04.
31 For the simulator running in [Gazebo](http://gazebosim.org/) a machine with (any) NVIDIA GPU has been proven useful.
32 For the object detection using [faster R-CNN](https://github.com/rbgirshick/py-faster-rcnn) a GPU with at least 2GB RAM is required.
33 For the object detection using [R-FCN](https://github.com/Orpine/py-R-FCN) and object segmentation using [MNC](https://github.com/daijifeng001/MNC) a GPU with at least 7 GB RAM are required.
34 We made good experiences with NVIDIA Quadro K2200 and NVIDIA TITAN X GPUs.

70 ## Known Limitations and Bugs
