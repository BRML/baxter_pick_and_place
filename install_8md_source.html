The Human-Baxter Collaboration Framework  v2.0
The Human-Baxter collaboration framework is a modular, easy to modify and adapt framework for collaborative experiments with human collaborators and a Baxter research robot.

1 # Installing and Preparing the Human-Baxter Collaboration Framework
2 
3 > The following instructions are adapted in parts from [here](http://sdk.rethinkrobotics.com/wiki/Workstation_Setup).
4 
5 
6 The Human-Baxter collaboration framework is implemented as a ROS package.
7 It has been tested with a development workstation with [Ubuntu 14.04](http://releases.ubuntu.com/14.04/) and [ROS Indigo](http://wiki.ros.org/indigo).
8 
9 > Note: If you have Ubuntu, ROS and and the Baxter SDK dependencies already
10 > installed, you only need to perform steps 3, 5 and 6 to clone, install and
11 > setup the Human-Baxter collaboration framework!

70 To install the Microsoft Kinect V2 for Linux we need the
71 [libfreenect2](https://github.com/OpenKinect/libfreenect2/blob/master/README.md#linux) library, which we will build from source and install into `~/freenect2`:
72 ```bash
73 $ sudo apt-get install libturbojpeg libopenni2-dev
74 $ git clone https://github.com/OpenKinect/libfreenect2.git ~/libfreenect2
75 $ cd ~/libfreenect2/depends
76 $ ./download_debs_trusty.sh
77 $ sudo dpkg -i debs/libglfw3*deb
78 $ sudo apt-get install -f
79 $ cd ~/libfreenect2
80 $ mkdir build
81 $ cd build
82 $ cmake .. -DENABLE_CXX11=ON -DCMAKE_INSTALL_PREFIX=$HOME/freenect2
83 $ make
84 $ make install
85 $ sudo cp ../platform/linux/udev/90-kinect2.rules /etc/udev/rules.d/
86 ```

90 ## Step 5: Install this package and its dependencies
91 
92 Using the [wstool](http://wiki.ros.org/wstool) workspace tool, you will checkout all required Github repositories into your ROS workspace source directory.
93 The [ROS bridge for the Kinect V2](https://github.com/code-iai/iai_kinect2#install) will be installed separately.
94 ```bash
95 $ cd $WS_HBCF/src
96 $ wstool init .
97 $ wstool merge https://raw.githubusercontent.com/BRML/baxter_rosinstall/master/baxter_pnp.rosinstall
98 $ wstool update
99 # now install dependencies for iai_kinect2
100 $ cd iai_kinect2
101 $ source /opt/ros/indigo/setup.bash
102 $ rosdep install -r --from-paths .
103 $ cd $WS_HBCF
104 $ catkin_make -DCMAKE_BUILD_TYPE="Release"
105 $ catkin_make install
106 ```
107 **Note**: `rosdep` will output errors about not being able to locate `[kinect2_bridge]` and `[depth_registration]`.
108 That is fine, since those are part of the `iai_kinect2` package.
109 
110 We now install [R-FCN](https://github.com/Orpine/py-R-FCN#requirements-software) for object detection using deep neural networks into `$DEVEL` (default in the framework settings is `~/software`).
111 Please refer to the [Caffe installation instructions](http://caffe.berkeleyvision.org/installation.html) if anything is unclear.
112 ```bash
113 $ mkdir $DEVEL
114 $ cd $DEVEL
115 # clone R-FCN and Microsoft Caffe commit 1a2be8e
116 $ git clone https://github.com/Orpine/py-R-FCN.git
117 $ cd $DEVEL/py-R-FCN
118 $ git clone https://github.com/Microsoft/caffe.git
119 $ cd $DEVEL/py-R-FCN/caffe
120 $ git reset --hard 1a2be8e
121 # build the Cython modules
122 $ cd $DEVEL/py-R-FCN/lib
123 $ make
124 # with your Makefile.config in place, build Caffe
125 $ cd $DEVEL/py-R-FCN/caffe
126 $ make -j8 && make pycaffe
127 ```
128 **Note**: Caffe *must* be built with support for Python layers!
129 That is, in your `Makefile.config`, make sure to have this line uncommented
130 ```
131 WITH_PYTHON_LAYER := 1
132 ```
133 
134 Optionally, [faster R-CNN](https://github.com/rbgirshick/py-faster-rcnn#requirements-software) and [MNC](https://github.com/daijifeng001/MNC#installation-guide) can be installed in a quite similar fashion.
135 Please refer to the respective installation instructions for details.

138 ## Step 6: Configure Baxter communication/ROS workspace
139 
140 The [baxter.sh](http://sdk.rethinkrobotics.com/wiki/Baxter.sh) script is a convenient script which allows for intuitive modification of the core ROS environment components.
141 This user edited script will allow for the quickest and easiest ROS setup.
142 Further information and a detailed description is available on the [baxter.sh](http://sdk.rethinkrobotics.com/wiki/Baxter.sh) page.
143 
144 
145 ### Download the baxter.sh script
146 
147 ```bash
148 $ cd $WS_HBCF
149 $ wget https://github.com/RethinkRobotics/baxter/raw/master/baxter.sh
150 $ chmod u+x baxter.sh
151 ```
152 
153 
154 ### Customize the baxter.sh script
155 
156 Using your favorite editor, edit the baxter.sh shell script making the necessary modifications to describe your development workstation.
157 
158 - Edit the `baxter_hostname` field to match the hostname of your Baxter robot.
159 - Edit **either** the `your_ip` **or** the `your_hostname` field to match the IP or hostname of your development workstation.
160 Only one of those fields can be active at a time.
161 The other variable should be commented out!
162 
163 
164 ### Initialize your SDK environment
165 
166 ```bash
167 $ cd $WS_HBCF
168 $ . baxter.sh
169 ```
170 
171 ### Verify environment
172 
173 To verify that all your changes are applied correctly, perform
174 ```bash
175 $ env | grep ROS
176 ```
177 The important fields at this point are
178 
179 - **ROS_MASTER_URI** (this should now contain your robot's hostname)
180 - **ROS_IP** or **ROS_HOSTNAME** (this should now contain your development workstation's ip address or hostname.
181 The unused field should **not** be available!)
